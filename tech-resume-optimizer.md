# Tech Resume Optimizer

## When to Use This Skill

Use when user is applying for SWE, PM, data science, DevOps, or other technical roles, or mentions "tech resume", "software engineer resume", "developer resume", "SWE resume", "PM resume".

## Tech Resume Structure

Recommended order:
1. Contact Information (including GitHub, portfolio URL)
2. Technical Skills (critical for ATS — before experience)
3. Work Experience (with technical achievements)
4. Projects (especially for early career)
5. Education
6. Certifications

Contact section should include: GitHub (required for SWE), portfolio/personal website, LinkedIn, tech blog if you have one.

## Technical Skills Section

**Option 1: By Category (recommended)**
```
Languages: Python, JavaScript, TypeScript, Go, SQL
Frameworks: React, Node.js, Django, FastAPI
Databases: PostgreSQL, MongoDB, Redis, Elasticsearch
Cloud/Infrastructure: AWS (EC2, S3, Lambda, RDS), Docker, Kubernetes, Terraform
Tools: Git, JIRA, CI/CD, Datadog, Grafana
```

**Option 2: Flat List (ATS-friendly)**
```
Skills: Python, JavaScript, React, Node.js, PostgreSQL, AWS, Docker, Git
```

Don't include: Microsoft Office, operating systems, skill bars or ratings, every technology you've touched once, outdated tech (unless specifically required).

## Technical Bullet Formula

**[Action Verb] + [Technical What] + [Scale/Impact] + [Technology Used]**

Weak: "Worked on backend services" / "Helped improve system performance"

Strong: "Architected microservices migration from monolith, reducing deployment time from 2 hours to 15 minutes" / "Optimized PostgreSQL queries and implemented Redis caching, reducing API latency by 60% (500ms to 200ms) for 100K DAU"

## Technical Metrics

Scale: "serving 500K DAU", "handling 10K requests/second", "processing 50TB daily", "99.99% availability"

Performance: latency (Xms to Yms), speed (improved by X%), load time (decreased X seconds)

Efficiency: cost ("reduced AWS costs by 40%"), time ("cut deployment time from X to Y")

Business: revenue driven, conversion improved, engagement increased

## Role-Specific Examples

**SWE:** "Designed authentication service using OAuth 2.0 and JWT, securing 2M+ accounts with zero security incidents"

**Data Engineer:** "Built data pipeline processing 100M+ events daily using Kafka and Spark, reducing data latency from hours to minutes"

**DevOps/SRE:** "Implemented IaC with Terraform, reducing provisioning time from 2 days to 30 minutes"

**Technical PM:** "Led API platform roadmap for developer tools used by 10K+ developers, driving 40% increase in API adoption"

## Projects Section

Critical for: junior engineers, career changers, bootcamp graduates, anyone with gaps.

Format:
```
Project Name | Technologies | github.com/link
• What it does + technical highlights
• Scale or usage metrics
```

Good projects: real users, open source contributions, complex personal projects, hackathon winners, technical blog posts.

## GitHub Profile

Pin your best 6 repos, maintain green contribution graph, add profile README, write good READMEs for each project (what it does, tech used, how to run, screenshots/demos).

## ATS + Tech Recruiter Balance

For ATS: exact skill keywords, standard section headers, avoid tables and graphics.
For Tech Recruiters: technical depth, metrics and scale, demonstrate problem-solving, show you understand systems.

## Handling Tech Stack Mismatch

If you match: lead with those technologies, quantify your experience.
If you don't match exactly: emphasize transferable skills, show learning ability, highlight similar technologies. Example: "Extensive Python web framework experience (Django); quick to ramp on new frameworks."
